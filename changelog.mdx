## Overview

This changelog tracks all updates, new features, and breaking changes to the Ramp Developer API. We recommend subscribing to our developer newsletter to stay informed about upcoming changes.

## June 24, 2024

### API Updates and Improvements

<CardGroup cols={2}>
  <Card title="Filter Purchase Orders" icon="filter">
    Use <code>spend_start_date</code> and <code>spend_end_date</code> to retrieve POs tied to a specific spend window.
  </Card>
  <Card title="Mileage Reimbursements" icon="car">
    New endpoint: <code>POST /reimbursement/mileage</code> — Log and submit mileage reimbursements via API.
  </Card>
  <Card title="Transaction Filtering" icon="clock">
    Filter transactions by update timestamp — <code>GET /transactions</code> now supports the <code>updated_after</code> filter for incremental syncs.
  </Card>
  <Card title="Search by Remote ID" icon="search">
    Search accounting vendors and GL accounts by <code>remote_id</code> — Simplifies mapping to ERP systems.
  </Card>
  <Card title="Bill-PO Matching" icon="link">
    Match bills to purchase orders — Enables programmatic linkage between bills and POs.
  </Card>
  <Card title="Manager Status" icon="user">
    Include <code>is_manager</code> in user invites — Supports setting manager status when inviting a user via API.
  </Card>
  <Card title="PO Filtering" icon="filter">
    Filter purchase orders by <code>user_id</code> — Use <code>GET /purchase-orders</code> with <code>user_id</code> to retrieve POs submitted by a specific user.
  </Card>
  <Card title="Accounting Date" icon="calendar">
    <code>accounting_date</code> now returned in reimbursements — Available via <code>GET /reimbursements</code> for improved reconciliation workflows.
  </Card>
  <Card title="Receipt Submission" icon="file-upload">
    New endpoint: <code>POST /reimbursements/submit-receipt</code> — Attach receipts directly to reimbursements via API.
  </Card>
  <Card title="OCR Auto-matching" icon="magic">
    Auto-match uploaded receipts using OCR — <code>POST /receipts</code> can now automatically associate receipts with matching transactions.
  </Card>
  <Card title="Temporary Limits" icon="chart-line">
    Temporary limit increases — Use <code>PATCH /limits/{limit_id}</code> to implement temporary spending limit boosts.
  </Card>
  <Card title="Declined Transactions" icon="eye-slash">
    Control declined transaction visibility — <code>GET /transactions</code> now defaults to excluding declines. Use <code>state=DECLINED</code> or <code>state=ALL</code> to include them.
  </Card>
  <Card title="Accounting Field Search" icon="search">
    Search by accounting field selections — Filter transactions and bills based on assigned accounting fields for better categorization.
  </Card>
  <Card title="Custom Forms" icon="file-text">
    Custom forms support — Retrieve and manage custom form metadata on Vendors, Purchase Orders, Transactions, and Limits using GET endpoints.
  </Card>
</CardGroup>

## April 30, 2024

### Deprecation Notice: Cards API

<Warning>
  **Deprecated**: Ramp is transitioning from the Cards API to the more flexible Limits API
  
  Use the Limits API for card issuance and management functionality
  
  The Limits API enables one-to-many relationships between limits and cards and shared limits across users
  
  Physical card shipping will continue to be supported via the Cards API until full migration support is provided
</Warning>
